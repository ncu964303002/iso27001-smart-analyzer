## 🎯 功能簡介

本工具是一個基於網頁的資安事件分析系統，採用智能關鍵字匹配引擎，能夠：

- ✅ **開放式輸入**：接受任何形式的資安事件描述
- ✅ **智能分析**：自動識別違反的 ISO 27001:2022 條文
- ✅ **風險評估**：評估事件的嚴重程度（高/中/低）
- ✅ **改善建議**：提供具體可執行的改善措施
- ✅ **離線可用**：完全在瀏覽器本地運作，無需網路

### 適用對象

- 👨‍💼 **資訊安全管理人員**：快速評估資安事件
- 🔍 **稽核人員**：檢查控制措施是否符合 ISO 27001
- 👨‍🏫 **教育訓練講師**：提供教學案例與說明
- 👥 **一般員工**：提升資安意識，了解事件影響
- 🏢 **資安顧問**：輔助客戶進行資安評估

## 🔍 功能詳解

### 1. 智能輸入驗證

**目的**
防止非資安問題被誤判，提升分析品質。

**驗證機制**
- 檢查輸入是否包含資安相關關鍵字
- 涵蓋 80+ 個中英文關鍵字
- 支援 ISO 27001 條文編號識別

**驗證結果**

✅ **通過驗證**（資安問題）
```
輸入：員工共用密碼
結果：進入分析流程
```

❌ **未通過驗證**（非資安問題）
```
輸入：今天天氣很好
結果：顯示友善提示
      說明工具適用範圍
      提供資安事件範例
```

**關鍵字類別**
- 核心詞彙：資安、安全、風險、漏洞...
- 技術詞彙：密碼、加密、防火牆、備份...
- 攻擊類型：駭客、病毒、釣魚、SQL Injection...
- 設備相關：電腦、伺服器、USB、手機...
- 管理流程：稽核、訓練、政策、離職...

### 2. 知識庫匹配引擎

**運作原理**
```
使用者輸入
    ↓
關鍵字匹配（80+ 關鍵字）
    ↓
識別相關資安領域（15 個類別）
    ↓
提取對應的 ISO 條文、風險、措施
    ↓
合併並去重
    ↓
生成分析結果
```

**15 個資安領域**

| 領域 | 關鍵字範例 | ISO 條文範例 |
|------|-----------|-------------|
| 密碼管理 | 密碼、帳號、登入 | A.5.17, A.5.18 |
| 存取權限 | 權限、授權、離職 | A.5.18, A.8.2 |
| 加密保護 | 加密、機密、敏感 | A.8.24 |
| 備份復原 | 備份、還原、勒索 | A.8.13, A.8.14 |
| 實體安全 | 機房、門禁、遺失 | A.7.2, A.7.4 |
| 網路安全 | 防火牆、攻擊、VPN | A.8.20, A.8.21 |
| 系統更新 | 更新、修補、漏洞 | A.8.8 |
| USB 管制 | USB、隨身碟 | A.8.1 |
| 雲端服務 | 雲端、Cloud | A.5.23 |
| 教育訓練 | 訓練、意識、釣魚 | A.6.3 |
| 日誌記錄 | 日誌、記錄、稽核 | A.8.15 |
| 供應商 | 供應商、廠商、委外 | A.5.19, A.5.20 |
| 事件應變 | 事件、應變、通報 | A.5.24, A.5.25 |
| 行動裝置 | 手機、BYOD、平板 | A.6.7, A.8.1 |
| 網站安全 | SQL、XSS、網站 | A.8.25, A.8.26 |

**匹配邏輯**
1. 將輸入轉為小寫
2. 逐一比對關鍵字
3. 記錄所有匹配的領域
4. 合併所有領域的資訊
5. 去除重複項目

### 3. 風險嚴重程度評估

**評估標準**

🔴 **高風險**
- 可能導致資料外洩
- 系統遭受攻擊
- 重大法規違規
- 業務嚴重中斷

🟡 **中風險**
- 控制措施缺失
- 管理流程不完善
- 潛在安全隱患
- 需要改善但非急迫

🟢 **低風險**
- 輕微的程序問題
- 建議性改善項目

**自動評估邏輯**
```javascript
如果匹配到的領域包含：
  - 密碼、加密、網路、更新、網站 → 高風險
  - 其他領域 → 中風險
  
沒有匹配任何領域 → 中風險（建議完整評估）
```

### 4. 改善措施建議

**措施特色**
- ✅ 具體可執行（非抽象概念）
- ✅ 符合 ISO 27001 要求
- ✅ 包含技術與管理面向
- ✅ 提供實施參考（如：時間、頻率）

**措施範例**
```
不好的建議：
❌ 加強密碼安全
❌ 提升員工意識

好的建議：
✅ 實施密碼複雜度政策（至少12字元，包含大小寫、數字、符號）
✅ 建立密碼定期更換機制（一般帳號90天、特權帳號60天）
✅ 每年至少舉辦 2 次全員資安教育訓練
```

## 💡 使用範例

### 範例 1：密碼管理問題

**輸入**
```
發現多位員工共用同一組系統管理員帳號密碼
```

**輸出**
```
📋 發現多位員工共用同一組系統管理員帳號密碼 [ 高風險 ]

🎯 識別關鍵字：
[ 員工 ] [ 共用 ] [ 系統 ] [ 管理員 ] [ 帳號 ] [ 密碼 ]

⚖️ 違反的 ISO 27001 條文
[ A.5.17 認證資訊 ]
[ A.5.18 存取權限 ]
[ A.8.2 特權存取權限 ]

⚠️ 風險影響分析
• 未授權人員可能取得系統存取權限
• 無法追蹤個人操作行為與責任歸屬
• 特權帳號濫用可能造成系統性破壞
• 違反最小權限原則，權限管理失控

✅ 建議改善措施
• 禁止帳號共用，每人必須使用個人專屬帳號
• 實施密碼複雜度政策（至少12字元）
• 特權帳號實施雙人授權機制
• 部署多因素認證(MFA)
• 記錄所有特權操作並定期審查
```

### 範例 2：資料保護問題

**輸入**
```
筆記型電腦遺失，內有未加密的客戶個資
```

**輸出**
```
📋 筆記型電腦遺失，內有未加密的客戶個資 [ 高風險 ]

🎯 識別關鍵字：
[ 筆電 ] [ 遺失 ] [ 未加密 ] [ 客戶 ] [ 個資 ]

⚖️ 違反的 ISO 27001 條文
[ A.8.24 密碼學使用 ]
[ A.7.2 實體進入控管 ]
[ A.5.10 資訊及其他相關資產之可接受使用 ]

⚠️ 風險影響分析
• 資料傳輸或儲存過程中可能被竊聽或竊取
• 違反個資法、GDPR等法規要求
• 機密資訊外洩導致商業損失或法律責任
• 實體媒體遺失或竊取

✅ 建議改善措施
• 筆記型電腦與行動裝置啟用全磁碟加密
• 敏感資料必須加密儲存（AES-256或以上）
• 筆記型電腦使用鎖鏈固定或鎖櫃保管
• 實施清桌政策，敏感文件不得放置桌面
• 設備報廢前必須進行資料銷毀
```

## ❓ 常見問題

### 一般問題

**Q1：這個工具需要付費嗎？**
A：完全免費，無任何隱藏費用。

**Q2：需要註冊帳號嗎？**
A：不需要，開啟即可使用。

**Q3：資料會被記錄或上傳嗎？**
A：不會。所有分析都在您的瀏覽器本地完成，不會傳送任何資料到外部伺服器。

**Q4：需要網路連線嗎？**

**Q8：可以分析英文輸入嗎？**
A：可以！系統支援中英文關鍵字識別。
